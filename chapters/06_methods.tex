\chapter{Methodology}

This chapter describes the theoretical approaches, methods, and algorithms used in each of the three studies comprising this thesis.

\section{Study 1: Uncertainty Estimation via Token-Wise Entropy}

\subsection{Entropy Extraction}

For each generated token $y_t$ in a model's response, we extract the full probability distribution over the vocabulary and compute the Shannon entropy:
\begin{equation}
    H_t = -\sum_{v \in \mathcal{V}} p_t(v) \log_2 p_t(v)
\end{equation}
where $p_t(v) = p(v | x, y_1, \ldots, y_{t-1})$ is the probability assigned to token $v$ at position $t$.

\subsection{Uncertainty Estimation Approaches}

We consider two approaches to uncertainty estimation: entropy-based and Model-as-Judge (MASJ).

\textbf{Single-token answer entropy.} For MMLU-Pro questions with multiple-choice answers, the expected answer is one or two tokens long. We prompt the model to answer directly (without chain-of-thought) and compute the token-wise entropy of the answer token from the model's output logits. This serves as our primary complexity metric, as response uncertainty is a natural predictor of question complexity.

\textbf{Chain-of-thought entropy variants.} We also analyze entropy when the model produces a chain-of-thought response. From the CoT response, we compute several aggregation features:
\begin{enumerate}
    \item \textbf{CoT Mean}: mean entropy across all tokens in the reasoning chain
    \item \textbf{CoT Max}: maximum entropy in the reasoning chain
    \item \textbf{Sequence-aggregation methods}: mean/max aggregated at the word level
    \item \textbf{Marginal difference}: mean difference between top-2 token probabilities
\end{enumerate}

\subsection{Model-as-Judge (MASJ) Approach}

As a complementary approach, we use a large auxiliary LLM (Mistral-Large-Instruct-2411, 123B parameters) to estimate question properties:
\begin{itemize}
    \item \textbf{Reasoning requirement}: whether the question requires reasoning and how many steps (low, medium, high)
    \item \textbf{Education level}: estimated difficulty as educational level (high school, undergraduate, graduate, postgraduate)
\end{itemize}

The MASJ scores are obtained via specific prompts and filtered using an MT-Bench-style quality assessment: instances where the judge assigns a score below 8 on a scale of 1 to 10 are regenerated.

\subsection{Evaluation Protocol}

We compare uncertainty estimates with ground-truth correctness labels using ROC-AUC, stratified by subject domain (14 topics), reasoning requirement (low/medium/high), and model architecture. Answers with incorrect formatting (less than 5\% of questions) are discarded.

\section{Study 2: Language Steering in Latent Space}

\subsection{Hidden State Extraction}

For a multilingual model $\mathcal{M}$ and a parallel corpus $\mathcal{D} = \{(s_i, k_i)\}_{i=1}^N$ where $s_i$ represents semantically equivalent content in language $k \in \mathcal{K}$, we extract hidden states $h_i^{(\ell)} \in \mathbb{R}^d$ from each layer $\ell$ of the LLM. Specifically, we use the hidden state at the \textbf{last token position} of each sentence. We use the Flores Plus dataset~\cite{nllb2024} with 50 parallel samples across languages (English, Spanish, Russian, Chinese, Hindi) for PCA fitting and 100 for validation.

\subsection{Language Direction Identification via PCA}

We apply Principal Component Analysis (PCA) to the collected hidden states to identify language-specific directions:

\begin{enumerate}
    \item \textbf{Data preparation}: Collect hidden states from the same layer for texts in two languages $\ell_1$ and $\ell_2$
    \item \textbf{Centering}: Compute the mean hidden state $\mu = \frac{1}{|H|}\sum_{h \in H} h$ and center the data
    \item \textbf{PCA}: Compute the principal components of the centered data
    \item \textbf{Direction extraction}: The first principal component $v_1$ typically captures the language direction
\end{enumerate}

The language direction $v_{\text{lang}}$ satisfies:
\begin{equation}
    v_{\text{lang}} = \arg\max_{\|v\|=1} \text{Var}(v^\top H)
\end{equation}

\subsection{Language Classification}

To validate that the identified direction captures language information, we train a linear classifier on the projection of hidden states onto the principal components:
\begin{equation}
    \hat{\ell} = \text{sign}(v_{\text{lang}}^\top h + b)
\end{equation}
where $b$ is a learned bias term. Classification accuracy above 95\% indicates successful language direction identification.

\subsection{Activation Steering}

During inference, we modify the model's hidden states to reduce code-switching by projecting out the source language component. For each token's hidden state $h_l^{(t)}$ at layer $l$, we apply:
\begin{equation}
    \tilde{h}_l^{(t)} = h_l^{(t)} - s \left(h_l^{(t)} \cdot v^{(l)}\right) v^{(l)}
\end{equation}

The parameter $s \in \mathbb{R}$ controls intervention strength. When $s = 1$, this is an exact orthogonal projection that removes the language component. Values $0 < s < 1$ partially attenuate it, while $|s| > 1$ goes beyond projection removal, actively pushing representations away from the source language direction. In practice, optimal coefficients have $|s| > 1$, indicating that mere removal is insufficient and active counter-steering is needed to overcome the language bias in code-switched input.

Steering is applied only to the final layers where language variance concentrates: for Qwen2.5-1.5B (28 layers), layers 27--28; for Llama-3.2-1B (16 layers), layers 15--16. The optimal steering coefficient $s$ is selected via grid search to minimize KL divergence between the steered model's output distribution and the target language baseline.

\subsection{Evaluation Metrics}

We evaluate steering effectiveness using two complementary evaluation approaches:

\textbf{Distributional evaluation.} We compare the next-token prediction distribution of the steered model against the English baseline using four metrics:
\begin{enumerate}
    \item \textbf{KL divergence} $\text{KL}(P_{\text{EN}} \| P_{\text{steered}})$: lower divergence indicates better mitigation
    \item \textbf{Jensen-Shannon divergence}: symmetric measure of distributional similarity
    \item \textbf{Cosine similarity}: between steered and English baseline distributions (higher is better)
    \item \textbf{Top-50 token overlap}: fraction of shared high-probability tokens with the English baseline
\end{enumerate}

\textbf{Generation-based evaluation.} We generate 100-token continuations and measure code-switching using standard metrics:
\begin{enumerate}
    \item \textbf{Code-Switching Index (CSI)}: fraction of tokens not in the target language (lower is better)
    \item \textbf{M-Index}: multilingual diversity measure ($0 = $ monolingual, $1 = $ maximally mixed)
    \item \textbf{I-Index}: ratio of language switch points to token boundaries
    \item \textbf{Target Language Consistency (TLC)}: $1 - \text{CSI}$ (higher is better)
\end{enumerate}
Token-level language identification uses a FastText classifier with a sliding window of 5 tokens.

\section{Study 3: Complexity-Aware Fine-Tuning}

\subsection{Complexity Estimation}

For each training example $(x, y)$, we estimate complexity by running the base model on $x$ and measuring the entropy of predicting $y$:
\begin{equation}
    C(x, y) = \frac{1}{|y|} \sum_{t=1}^{|y|} H(p(\cdot | x, y_{<t}))
\end{equation}

High complexity indicates that the model is uncertain about the correct response, suggesting the example is difficult.

\subsection{Data Stratification}

We partition the training data into three equal complexity tiers by sorting examples by entropy in ascending order:
\begin{align}
    \mathcal{D}_{\text{easy}} &= \mathcal{D}[1 : \lfloor N/4 \rfloor] \\
    \mathcal{D}_{\text{medium}} &= \mathcal{D}[\lfloor N/4 \rfloor : \lfloor 3N/4 \rfloor] \\
    \mathcal{D}_{\text{hard}} &= \mathcal{D}[\lfloor 3N/4 \rfloor + 1 : N]
\end{align}
The easy and medium examples are combined into a ``regular'' group $\mathcal{D}_{\text{regular}} = \mathcal{D}_{\text{easy}} \cup \mathcal{D}_{\text{medium}}$ that receives standard SFT, while hard examples receive chain-of-thought distillation.

\subsection{Training Interventions}

Different complexity tiers receive different training treatments:

\textbf{Easy examples} ($C < \tau$): Direct supervised fine-tuning (SFT) with the standard cross-entropy loss:
\begin{equation}
    \mathcal{L}_{\text{SFT}}(x, y) = -\sum_{t=1}^{|y|} \log p_\theta(y_t | x, y_{<t})
\end{equation}

\textbf{Hard examples} ($C \geq \tau$): Chain-of-thought distillation from a larger teacher model $\mathcal{M}_T$:
\begin{enumerate}
    \item Generate reasoning chains $r$ from the teacher: $r \sim \mathcal{M}_T(\cdot | x, \text{``think step by step''})$
    \item Train the student on the augmented examples:
    \begin{equation}
        \mathcal{L}_{\text{distill}}(x, y) = -\sum_{t=1}^{|r|+|y|} \log p_\theta((r \oplus y)_t | x, (r \oplus y)_{<t})
    \end{equation}
\end{enumerate}
where $r \oplus y$ denotes concatenation of the reasoning chain and the answer.

\subsection{Training Pipeline}

The complete training pipeline consists of:
\begin{enumerate}
    \item \textbf{Complexity scoring}: Run the base model on all training examples to compute complexity scores
    \item \textbf{Stratification}: Partition data into easy and hard subsets based on threshold $\tau$
    \item \textbf{Teacher generation}: For hard examples, generate chain-of-thought reasoning from the teacher model
    \item \textbf{Mixed training}: Fine-tune the student model on the combined dataset with appropriate losses
    \item \textbf{Evaluation}: Assess accuracy on held-out test data
\end{enumerate}

\subsection{Hyperparameter Selection}

Key hyperparameters include:
\begin{itemize}
    \item Complexity metric: Single-token answer entropy (selected based on ROC-AUC comparison with alternatives including MASJ reasoning score and cross-entropy)
    \item Teacher model: An ensemble of large models (DeepSeek-V3-0324, Qwen-3-235B, Llama-4-Maverick) for chain-of-thought generation
    \item Training schedule: SFT on regular data for 5 epochs, followed by SFT with reasoning-enriched data for hard examples for another 5 epochs
    \item Fine-tuning method: LoRA with rank 64, alpha 128, dropout 0.05
\end{itemize}

\section{Implementation Details}

All experiments are implemented in Python using the PyTorch framework~\cite{paszke2019pytorch} and the Hugging Face Transformers library~\cite{wolf2020transformers}. Hidden state extraction and entropy computation are performed during standard forward passes without requiring gradient computation. Activation steering is implemented using forward hooks that modify hidden states in-place during generation.
